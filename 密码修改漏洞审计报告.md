### 🔴 VULN-01：越权修改密码

#### 漏洞信息

| 项目 | 内容 |
|:-----|:------|
| **漏洞类型** | 越权操作（CWE-287: Improper Authentication） |
| **严重等级** | 🔴 严重 |
| **CVSS 3.0** | 9.1（AV:N/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:N） |
| **发现位置** | `app.py` - `/change-password` 路由，第378行 |

#### 漏洞描述

修改密码接口从表单接收 `username` 参数，直接使用该参数更新密码，**未验证当前登录用户和要修改的 username 是否一致**。任何已登录用户都可以修改任意其他用户的密码。

更严重的是，`<input type="hidden" name="username" value="{{ user.username }}">` 虽然默认是当前用户，但攻击者可以通过**修改页面源码**或**直接构造 POST 请求**来提交任意用户名。

#### 攻击场景

```
攻击者以 alice 身份登录（普通用户，余额100元）

请求：POST /change-password
参数：username=admin&new_password=hacked123

结果：管理员密码被改为 hacked123
攻击者使用 hacked123 登录 admin，获取所有权限
```

#### 代码问题分析

**修复前：**
```python
@app.route('/change-password', methods=['POST'])
def change_password():
    username = request.form.get('username', '')      # ❌ 来自用户输入
    new_password = request.form.get('new_password', '')
    
    pw_hash = bcrypt.hashpw(new_password.encode(), bcrypt.gensalt()).decode()
    c.execute("UPDATE users SET password_hash = ? WHERE username = ?", 
              (pw_hash, username))                   # ❌ 使用用户输入的username
```

**问题：**
| 行 | 问题 |
|:--:|:-----|
| `username = request.form.get('username', '')` | ❌ `username` 来自表单，用户可以随意修改 |
| `WHERE username = ?` | ❌ 使用外部传入的 `username` 更新密码 |

**修复后：**
```python
@app.route('/change-password', methods=['POST'])
@csrf_required
def change_password():
    if 'username' not in session:
        flash('请先登录', 'warning')
        return redirect(url_for('login'))

    old_password = request.form.get('old_password', '')
    new_password = request.form.get('new_password', '')

    # 验证原密码（从 session 获取用户，不从表单）
    c.execute("SELECT password_hash FROM users WHERE id = ?", (session['user_id'],))
    user = c.fetchone()
    if not user or not bcrypt.checkpw(old_password.encode(), user['password_hash'].encode()):
        flash('原密码错误', 'error')
        return redirect(url_for('profile', user_id=session.get('user_id')))

    # 使用 session 中的用户信息更新密码
    c.execute("UPDATE users SET password_hash = ? WHERE id = ?", 
              (pw_hash, session['user_id']))          # ✅ 从 session 取用户
```

### 🔴 VULN-02：无原密码校验

#### 漏洞信息

| 项目 | 内容 |
|:-----|:------|
| **漏洞类型** | 身份验证缺失 |
| **严重等级** | 🔴 严重 |
| **发现位置** | `app.py` - `/change-password` 路由 |

#### 漏洞描述

修改密码接口**没有要求输入原密码**，也没有对原密码进行校验。只要攻击者能接触到已登录用户的浏览器（或被 CSRF 攻击），就可以直接修改密码。

#### 攻击场景

```
场景1：临时借用电脑
1. 用户A借同事B的电脑登录了自己的账号
2. 忘记退出登录
3. 同事B打开个人中心，直接修改用户A的密码
4. 用户A无法再登录

场景2：CSRF组合攻击（结合VULN-03）
1. 管理员已登录系统
2. 攻击者诱导管理员访问恶意页面
3. 恶意页面自动提交：new_password=hacked
4. 管理员密码被改为 hacked（无需原密码）
```

#### 修复方案

必须验证原密码：从数据库取出当前用户的密码哈希，用 `bcrypt.checkpw()` 比对用户输入的原密码。

```python
# 从 session 获取当前用户
c.execute("SELECT password_hash FROM users WHERE id = ?", (session['user_id'],))
user = c.fetchone()

# 验证原密码
if not user or not bcrypt.checkpw(old_password.encode(), user['password_hash'].encode()):
    flash('原密码错误', 'error')
    return redirect(url_for('profile', user_id=session.get('user_id')))
```

### 🟡 VULN-03：修改密码无 CSRF 防护

#### 漏洞信息

| 项目 | 内容 |
|:-----|:------|
| **漏洞类型** | 跨站请求伪造（CWE-352: Cross-Site Request Forgery） |
| **严重等级** | 🟡 中 |
| **发现位置** | `app.py` - `/change-password` 路由 |

#### 漏洞描述

修改密码的 POST 请求**没有 CSRF Token 验证**，也没有要求携带任何防跨站标记。攻击者可以构造恶意 HTML 页面，在受害者不知情的情况下利用其登录态提交密码修改请求。

#### 攻击场景

```html
<!-- 攻击者构造的恶意页面 -->
<html>
<body>
    <h1>恭喜你中奖了！</h1>
    <form id="f" action="http://127.0.0.1:5000/change-password" method="POST">
        <input type="hidden" name="username" value="admin">
        <input type="hidden" name="new_password" value="attacker123">
    </form>
    <script>document.getElementById('f').submit();</script>
</body>
</html>
```

当已登录 admin 的用户访问此页面时：
```
1. 页面自动提交表单到 /change-password
2. 浏览器自动携带 admin 的 session Cookie
3. 密码被改为 attacker123
4. 用户完全不知情
```

#### 修复方案

添加 `@csrf_required` 装饰器，并在表单中加入 CSRF Token：

```python
# 后端添加装饰器
@app.route('/change-password', methods=['POST'])
@csrf_required                              # ✅ 新增
def change_password():
```

```html
<!-- 前端表单添加隐藏 Token -->
<form method="POST" action="/change-password">
    <input type="hidden" name="csrf_token" value="{{ csrf_token() }}">  <!-- ✅ 新增 -->
    ...
</form>
```
